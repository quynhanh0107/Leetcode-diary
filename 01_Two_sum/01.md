### 01. Two sum
Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

You can return the answer in any order.

 
Example 1:

Input: `nums = [2,7,11,15]`, `target = 9`
Output: `[0,1]`
Explanation: Because `nums[0] + nums[1] == 9`, we `return [0, 1]`.


Example 2:

Input: `nums = [3,2,4]`, `target = 6`
Output: `[1,2]`


Example 3:

Input: `nums = [3,3]`, `target = 6`
Output: `[0,1]`
 

Constraints:

2 <= `nums.length` <= 104
-109 <= `nums[i]` <= 109
-109 <= `target` <= 109
Only one valid answer exists.

### Solution explanation
- The use of `Dictionary` (in Python) or `HashMap` (in C++) are the optimal approach to this problem: it has a time complexity of $O(n)$. It only requires a single pass through the list, meaning a maximum of 10,000 checks. Meanwhile, the **nested loop** method would be slower with a time complexity of $O(n²)$. If your list has 10,000 numbers, the code might have to do up to 100,000,000 checks in the worst case.
- The basic idea for the second approach: Instead of searching for pairs, you look at the *current number*, calculate exactly what number you need to reach the target, and check if you've already seen it.
